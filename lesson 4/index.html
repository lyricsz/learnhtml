HTML Attributes

We've talked about HTML tags, elements and contents. We've also discussed about void element. (If you didn't know about these yet, kindly review the past lessons).

HTML Attributes in short adds additional information to a tag. You can also think of these informations as properties. Some tags comes with built in attributes that we can override. And some attribute... are just what we come up with.Web browsers did not understood Attributes that we come up with. (But we can use them with other languages such as CSS ---styling- and JavaScript ---Dynamic-.)

The first most important attribute we are going to look at is the "lang" attribute. Lang' is short for language. But first of all, where do we put attributes? At the opening tag for both void elements and other element. Take a look at the following template:

  • For other element:
    • <tagName attribut=attributeValue> content </tagName>
  • For void element:
    • <tagName attribute=attributeValue />

Now that we've seen the syntax of HTML attributes. Let's look at some examples. Starting with the lang' attribute.

Lang attribute is used to specify the language used in a webpage. If not specified web browser tends to guess by looking at the content of our webpage. This can create lots of hidden trouble. E.g our texts can be rendered in a very weird way we did not want. To fix this, specify the language used by adding a lang attribute to the ROOT opening tag. This is a simple example:
<!DOCTYPE html>
<html lang='en'>
<head>
<title> This website language is set to English </title>
</head>
<body>
</body>
</html>

Note

You might be tempted to overlook the lang attribute. This is not recommended. Your website might not be rendered properly in production. So always set the language for your webpage.

Voila. The above code now tells the web browser that "Please don't give me any other language. My language is English. ". The language attribute should be unique. Unless your page is an article with different language. (We'll discussed about this in the future. ) In short, the language you used in a webpage must be consistent. If you start with English, stick with English to the end.
Note

You can use both double quote and single quote for the attributeValue ðŸ˜‰. In other words both of the following are valid:

  • Single quote is valid
    • <tagName attribute='attributeValue' />
  • Double quote is valid
    • <tagName attribute="attributeValue" />

But since HTML is so forgiving ("that" has it's limit), the below code is also valid:

However, even though it is valid. Never use it. Never use it. It's 100% not recommended. Use single quote or double quote instead.

Are you kidding me, Tom. Just used non-breaking-space entities and your life will never be the same.

For a second... Imagine if you can write all the stickers, characters, in this world. but how's that even possible? HTML Entities is your answer. HTML Entities adds character to your webpage. This means that whatever you typed stands for something even if it's nothing like "spaces". It is something. Each entity represents a character. There are alot of possible characters available to you. They even range to stickers.

HTML Entities uses the following syntax:

  • Using entity name:
    • &entityName
  • Using entity number:
    • &#entityNumber
  • Using entity hexadecimal:
    • &#xentityHexadeciaml

The above syntax might look cryptic at first. As you use it, you get more familiar with it. You can use any of the three. I use entity name most of the time.

Note

If you use entity name, append (or add a prefix) of "&". If you use entity number (decimal 0 - 9 --- base 10), append (or add a prefix) of "&#". If you use entity hexadecimal (0 - 15 --- base 16), append (or add a prefix) of "&#x".

Let's look at some use cases of HTML Entities. Looking at my keyboard right now i couldn't find copyright symbol. So let's see how how i can add copyright to a webpage using entities.

<head>

</head>

The above code is incorrect. If you run it, it does works right? But always end your HTML Entities with ";". (Always follow the best practices ðŸ˜Š).

Let's look at some other two most important entities.

As we've discussed in past lessons, you can't add more than one spaces to your html code. Web browser will trimmed it to only a space. But what if i told you can use entities to achieve this goal? We use the non-breaking space entity. This entity is a little bit weird. It exist as if it doesn't exist. It is a space but it is a character. Hence you can add as many space to your webpage as possible. You can also use it to join two words together. ( i think an example will explain this better).

<!DOCTYPE html>
<html lang="en">
<head>
<title> A Non-breaking space </title>
</head>
<body>
&nbsp;&nbsp;&nbsp;&nbsp; Hello world. This line starts with four space.
(The following words are just one word to the web browser, hence they never seperate. They must all be in the same line. If the line is not enough for them, they create scrollbars. Try it out. ) &nbsp;There&nbsp;Lived&nbsp;an&nbsp;amazing&nbsp;web&nbsp;developer&nbsp;.&nbsp;
</body>
</html>

You can make the second example in the above code as long as possible, but they will never break as seperate words do... since you are using an entity instead of spaces.

The last most use case is when you want to add a less-than symbol to a webpage or a greater-than symbol. If you avoid entities, the web browser will automatically think you are starting a tag or closing a tag. But that's not the case. We only want to include them in a webpage. Let's take a look at below code:

<!DOCTYPE html>
<html lang="en">
<head>
<title> Did you know? </title>
</head>
<body>
&nbsp;&nbsp;&nbsp;&nbsp; Did you know?
20 + 20 &gt; 30
&nbsp;&nbsp;&nbsp;&nbsp; Did you know?
20 - 20 &lt; 30
</body>
</html>

Change the &lt; to < and &gt; to >, notice how the web browser completely misunderstood you? The below table summarizes all HTML entities we've discussed so far:

Entities name Usage Description
nbsp I&nbsp;Love&nbsp;You Used to add empty character to webpage. Appear the same as adding space to webpage. (nbsp stands for non-breaking space).
lt is 2 &lt; 4? Used to add less-than character to webpage. If normal less-than is used, web browser might think we meant something else. (lt stands for less-than).
gt is 2 &gt; 4? Used to add greater-than character to webpage. If normal greater-than is used, web browser might think we meant something else. (gt stands for greater-than).
copy &copy;2024 Used to add copyright character to webpage. Most keyboards don't have the character. (copy stands for copyright).
